Welcome to the issuer configurator and generator.



You can add an issuer to your code as a webhook.


What you need to do is to install in your code a webhook and a callback endpoint and redirect your user to a QR code page.
It takes no more than 2 hours.


How it works ?


When you want the user to collect her credential you redirect her browser to the issuer page provided by the platform so she can scan a QR code (or activate a deeplink).
Then your application receives a call on its webhook endpoint with eventually the requested credential of your choice in the form of a json web token.
Your application prepares the credential json data to issue and transfers it in response to the webhook call. The platform signs the credential and transfers it to the user's wallet.
If the transfer is sccessful the user is redirected to your application callback endpoint.
The credential is then accepted or rejected by the user and stored in the wallet.
Most of the parameters (DID method, keys, credentials, page design,...) can be configured on the platform in a couple of minutes.
