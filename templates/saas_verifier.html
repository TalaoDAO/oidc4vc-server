Welcome to the verifier generator and bridge



You can add a verifier to your code as an OpenID standard login similar to Facebook or Google Connect (or FranceConnect).


What you need to do is to install in your code a client OpenID library; it takes no more than 2 hours.


A list of libraries available for different platform is here.
An example of python code is there.

To activate the bridge you will need the following OpenID parameters :


Issuer : {{issuer}}
client_id : {{client_id}}
client_secret : {{client_secret}}


How it works ?


For each user you want to get a verifiable credential, you request an OpenID authentication.
The bridge will manage a user redirection to a page with a QR code (or a deeplink for mobile browser) allowing the user to present a verifiable credential stored in his wallet.
Your application will receive the user's credential in the form of a json file on the userinfo endpoint.
The verification of the signatures of the issuer and the user's wallet is carried out by the bridge, the application can simply access the attributes of the json received securely.
This verifier is initialized to receive any type of credential. You will be able then to update it or to generate several other different verifiers.
