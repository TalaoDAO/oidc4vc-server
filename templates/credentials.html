User credentials without identity disclosure

Blockchain address ownership
For web3 applications this is for sure the most important credential as it allows user to bind their decentralized identity to a blockchain address. Privacy is guaranteed because the private key of the decentralized identity is different from the private key of the blockchain address. The credential is issued by the non custodial user wallet at creation.
Phone proof
User can associate their phone number to their decentralized identity after having successfully passed an SMS test. The process is carried out by Altme. Data are stored in the user wallet. The validity is 1 year. See issuer of Phone proof here
Email proof
User can associate their email to their decentralized identity after having successfully passed an email test. The process is carried out by Altme. Data are stored in the user wallet. The validity is 1 year. See issuer of Email proof here
Discord account
This is a proof of the ownership of a Discord account, it can be bound to a community. The process is carried out by Altme. Data are stored in the user wallet.
Tweeter account
This is a proof of the ownership of a Tweeter account. The process is carried out by Altme. Data are stored in the user wallet.
Community reputation proof
This credential is issued by Altme on behalf of a community. User can access to a proof og their reputation in different communities. User can display this credential on a public website or a social network account or even transfer it to another user.
Ranking
This credential is issued by Altme on behalf of the operators. User can access to a proof og their rankings on different games. The credential is stored in the user wallet only. User can display their proof of ranking on a public website or a social network account even transfer it to another user.

User credentials with identity verification and limited disclosure

Nationality
User can associate their nationality to their decentralized identity with a KYC process embedded in the wallet. The process is carried out by a KYC provider (180 countries are covered), the credential is issued by Altme. Data are stored in the user wallet.
Gender
User can associate their decentralized identity to that credential with a KYC process embedded in the wallet. The process is carried out by a KYC provider, the credential is issued by Altme. Data are stored in the user wallet.
Age range
User can associate their age range (-18, 18-24, 25-34, 35-44, 45-54, 55-64, 64+) to their decentralized identity with a KYC process embedded in the wallet. The credential is issued by Altme. Data are stored in the user wallet. The validity is 1 year or less.
Proof of humanity
I am not a bot !
The verification process is carried out by a 3D facial liveness test embedded in the wallet. The credential is issued by Altme.
Over 18
User can associate their decentralized identity to that credential with a KYC process embedded in the wallet. The process is carried out by a KYC provider, the credential is issued by Altme. Data are stored in the user wallet.
Passport number footprint
The verification process is carried out by a KYC process embedded in the wallet. A hash is calculated from the passport MRZ (Machine-Readable Zone). The credential is issued by Altme. No identity disclosure.

User credentials with extended identity verification

Identity card - KYC
The KYC process is carried out through a 3D facial liveness test and an integrated document authentication process in the mobile device with prior biometric authentication of the user. Altme retains the KYC process ID for auditing. No personal data are stored by Altme.
Anti money laundering
User can associate their decentralized identity to a watchlist check carried out in the KYC provider backend. The credential is issued by Altme. Altme retains the KYC process ID for audit. No data are stored by Altme.
Proof of residency
User can associate their decentralized identity to a proof of residency with an extended document authentication process embedded in the wallet. The process is carried out by a KYC provider, the credential is issued by Altme. Altme retains the KYC process ID for audit. No data are stored by Altme.
